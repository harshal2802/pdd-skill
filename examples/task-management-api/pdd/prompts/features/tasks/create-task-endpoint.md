# Prompt: POST /api/v1/tasks

**File**: pdd/prompts/features/tasks/create-task-endpoint.md
**Created**: 2026-02-03
**Project type**: Backend / API

## Context

We are building TaskFlow, a task management REST API with Node.js/Express, TypeScript, PostgreSQL, and Prisma. The codebase follows a controller/service/repository layering pattern. Input validation uses Zod. Auth is JWT-based via `requireAuth` middleware.

The `users` table and auth endpoints already exist. This is the first endpoint in the task management domain.

## Task

Create the `POST /api/v1/tasks` endpoint that allows an authenticated user to create a new task. Implement the full vertical slice: Prisma schema addition, repository, service, controller, route, validator, and tests.

## Input

The endpoint receives a JSON request body:

```json
{
  "title": "Design the onboarding flow",
  "description": "Create wireframes for the first-time user experience.",
  "priority": "high",
  "due_date": "2026-03-15",
  "assigned_to": "uuid-of-user"
}
```

Field rules:
- `title` (string, required): 1-200 characters
- `description` (string, optional): max 2000 characters
- `priority` (enum, optional): `low`, `medium`, `high`, `urgent`. Defaults to `medium`.
- `due_date` (string, optional): ISO 8601 date, must be today or in the future
- `assigned_to` (UUID, optional): must reference an existing user

## Output format

Return these files:

1. `prisma/schema.prisma` — add the `Task` model (include `id`, `title`, `description`, `status`, `priority`, `due_date`, `assigned_to`, `created_by`, `created_at`, `updated_at`)
2. `src/repositories/task-repository.ts` — `create(data): Promise<Task>`
3. `src/services/task-service.ts` — `createTask(input, currentUserId): Promise<Task>` with validation logic
4. `src/validators/create-task.validator.ts` — Zod schema for the request body
5. `src/controllers/task-controller.ts` — parse request, validate, call service, format response
6. `src/routes/task-routes.ts` — wire `POST /` to the controller, apply `requireAuth`
7. `src/services/task-service.test.ts` — unit tests for the service (mock repository)
8. `tests/integration/create-task.test.ts` — integration tests with Supertest

## Constraints

- `id` must be a UUID, generated by the database (`uuid_generate_v4()` or Prisma `@default(uuid())`)
- `status` defaults to `open` and is NOT settable by the client on creation
- `created_by` is set from the authenticated user's JWT, never from the request body
- If `assigned_to` is provided, verify the user exists; return 404 if not
- Follow the error shape: `{ error: { code, message, details? } }`
- No raw SQL; use Prisma for all database operations
- No business logic in the controller; no Prisma calls in the service

## Examples

**Successful creation:**

Request:
```
POST /api/v1/tasks
Authorization: Bearer <valid-jwt>
Content-Type: application/json

{ "title": "Write unit tests for auth service" }
```

Response (`201 Created`):
```json
{
  "data": {
    "id": "a1b2c3d4-...",
    "title": "Write unit tests for auth service",
    "description": null,
    "status": "open",
    "priority": "medium",
    "due_date": null,
    "assigned_to": null,
    "created_by": "user-uuid-from-jwt",
    "created_at": "2026-02-03T10:00:00.000Z",
    "updated_at": "2026-02-03T10:00:00.000Z"
  }
}
```

**Validation error:**

Request:
```
POST /api/v1/tasks
Authorization: Bearer <valid-jwt>
Content-Type: application/json

{ "title": "", "priority": "critical" }
```

Response (`400 Bad Request`):
```json
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Invalid request body",
    "details": {
      "title": "Must be between 1 and 200 characters",
      "priority": "Must be one of: low, medium, high, urgent"
    }
  }
}
```
